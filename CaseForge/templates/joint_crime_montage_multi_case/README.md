# Joint Crime Montage Multi-Case Template

模板类型：共同犯罪 / 蒙太奇叙事 / 多案合一 / 因果误导推理

适用案件：Case004 及后续高难度复合型案件。

---

## 1. 模板目标

本模板用于开发一种比常规时间线推理更复杂、但仍保持公平性的案件。

玩家表面上面对多个案件：

```text
失踪案 / 死亡案 / 纵火案 / 伪证案 / 旧案重启
```

但最终应意识到：

```text
这些不是并列案件，而是同一核心事件在不同时间、不同视角、不同参与者手中形成的连锁后果。
```

本模板核心不是“谁独自杀了人”，而是：

```text
哪些人分别做了看似不致命的一步，最终共同完成了犯罪闭环？
```

---

## 2. 三个核心机制

### 2.1 共同犯罪

共同犯罪不等于多人一起动手杀人。

推荐结构：

```text
A 只制造机会。
B 只隐瞒时间。
C 只搬动物品。
D 只销毁证据。
E 只伪造不在场证明。
```

单看每一步，都不像完整犯罪。

但拼合后：

```text
缺任何一步，案件都不会成立。
```

### 2.2 蒙太奇误导

蒙太奇不是让 NPC 撒谎，而是让玩家自然把相邻片段拼成错误因果。

常用误导：

- 相邻段落并非连续时间。
- 同一句“他回来了”指向不同人物。
- 两段回忆来自不同日期，却被玩家理解成同一晚。
- 一个动作是结果，却被玩家误认为原因。
- 一个“案发后行为”其实发生在案发前。

### 2.3 多案合一

玩家前期应认为存在多个案件。

后期应能归并为：

```text
同一核心事件的前置、过程、遮掩、后果、二次激活。
```

---

## 3. 公平性硬规则

为避免 Case003 后期发散，本模板必须遵守：

1. 第一幕内登场全部核心人物。
2. 后续不新增关键嫌疑人。
3. 第一幕内出现全部关键物证的影子。
4. 物证不得撒谎。
5. NPC 可以误解、隐瞒、撒谎，但不能替作者临时补设定。
6. 结局不得依赖最后突然出现的新证据。
7. 真相必须能由玩家在中段提前推导。
8. 误导玩家的是片段顺序和因果解释，而不是缺失信息。

## 5. 玩家体验目标

前期：

- 玩家认为自己在处理多个案件。
- 每个案件都有独立嫌疑人。
- 证据之间存在微弱重复物。
- 叙事顺序自然诱导错误因果。

中期：

- 玩家开始发现多个案件共享同一物品、同一时间缺口、同一证人盲点。
- 玩家怀疑不是多案并列，而是多案同源。

后期：

- 玩家重排片段顺序。
- 玩家识别各 NPC 各自负责的“小罪”。
- 玩家完成共同犯罪链闭环。
