# Playtest Template

状态：TEMPLATE

> 本文件用于案件自测。目标不是证明案件完美，而是找出是否能跑通、哪里会卡、哪里存在逻辑漏洞。

---

## 1. Playtest 目标

- [ ] 玩家能正常开案。
- [ ] 玩家能获得基础现场信息。
- [ ] 玩家能发现核心证据链。
- [ ] 玩家能质询主要 NPC。
- [ ] 玩家能修正错误认知。
- [ ] 玩家能提交结案陈述。
- [ ] DM 能根据 ending.md 给出判定。
- [ ] 不出现致命逻辑断裂。

---

## 2. 测试路线 A：标准路线

1. 阅读开局简报。
2. 调查主要地点。
3. 检查核心物证。
4. 调取记录/监控/报告。
5. 询问主要 NPC。
6. 整理时间线。
7. 提交结案陈述。

通过标准：

- [ ] 能自然获得主线证据。
- [ ] 能自然修正主要误导。
- [ ] 结案判定可执行。

---

## 3. 测试路线 B：跳跃路线

模拟玩家一开始就调查强线索：

- [ ] 先查监控/记录。
- [ ] 先盘 NPC。
- [ ] 先查尸体/核心物。
- [ ] 先提交阶段推理。

通过标准：

- [ ] 不直接剧透。
- [ ] 不致命卡关。
- [ ] 仍需证据链闭合。

---

## 4. 测试路线 C：恶意/乱序路线

测试动作：

- [ ] 开局直接指认某人。
- [ ] 不查关键证据直接结案。
- [ ] 一直翻无关物品。
- [ ] 把所有人说成合谋。
- [ ] 反复追问无证据方向。

通过标准：

- [ ] DM 能提示证据不足。
- [ ] DM 不强行公布答案。
- [ ] 普通物品不会临时变成关键证据。
