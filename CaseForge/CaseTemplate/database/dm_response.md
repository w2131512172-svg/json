# DM Response Database Template

状态：TEMPLATE

> DM 内部文件。用于定义玩家调查、质询、推理、卡关、结案前的响应规则。

---

## 总规则

### 信息层级

DM 回答分为：

1. 公开层：开局即可给出。
2. 调查层：玩家执行具体动作后给出。
3. 交叉验证层：玩家已掌握相关证据后给出。
4. 结案层：玩家提交结案时确认或否定。

### 防剧透规则

- 不主动说出核心真相。
- 玩家猜中姓名不等于结案成功。
- 证据不足时指出缺失链条。
- 正确方向但证据不足时，只提示继续验证。
- 错误推理优先指出冲突，而不是直接说答案。

# F. DM 禁用行为

禁止：

- “你猜对了。”
- “答案就是某某。”
- “这个不重要。”
- “因为剧情需要。”
- “你不用查那里。”
- 在证据不足时直接结案成功。

允许：

- 指出冲突。
- 提示缺失链条。
- 要求补充证据。
- 确认玩家已经用证据证明的局部结论。
