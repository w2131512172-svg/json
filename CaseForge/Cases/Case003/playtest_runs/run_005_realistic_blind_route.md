# Run 005：真实玩家盲测模拟路线

案件：Case003《第三次葬礼》
版本：v0.3 Verified Playable Draft
测试类型：模拟真实玩家盲测
状态：SIMULATED
结果：SOFT PASS

---

## 1. 测试目标

验证 Case003 在接近真实玩家的游玩方式下是否能稳定运行。

重点检查：

1. 玩家是否会在前期被多线信息压垮。
2. 玩家是否会过早尝试整理标准时间轴。
3. 玩家是否能理解“碎片归组”比“时间排序”更重要。
4. 传闻线是否会误导过头。
5. 最终核验场景是否足以承接结案。
6. DM 是否需要临场补核心设定。

---

## 2. 模拟玩家行为设定

该玩家不是 Case002 那种“事件链优先”的结构型玩家。

模拟路线更接近普通高阶玩家：

```text
先追最有戏剧性的传闻
→ 盯住纸扎铺线
→ 被葬礼线拉回
→ 尝试排时间轴失败
→ 开始用碎片板归组
→ 通过旧款线进入因果层
→ 最终进入核验场景
```

---

## 3. 模拟调查流程

### Phase 1：传闻入口

玩家先调查：

- `player_materials/rumor_clippings.md`
- 镇口旧戏台
- 阿禾问话

玩家早期判断：

```text
本案可能围绕“死人回来”的传闻展开。
```

系统表现：

- 传闻材料能吸引玩家进入。
- 阿禾线能制造悬疑感。
- F06、F10、F14 足够支撑第一层误导。

问题：

玩家可能过度停留在传闻线，需要中期纠偏。

---

### Phase 2：纸扎铺误锁

玩家随后调查：

- 林小满
- 无名纸人
- 旧外套
- 夜里敲门的人

玩家形成错误假设：

```text
纸扎铺可能是核心。
林小满可能知道很多，甚至可能是主动推动者。
```

系统表现：

- F04、F09、F13 的误导强度足够。
- 但 `final_questions.md` 和 DM 响应能在后期把她从“主谋位”降回“关键视角位”。

问题：

如果玩家极端执着该线，仍可能卡一段时间。

---

### Phase 3：葬礼线回拉

玩家调查：

- 祠堂白布
- 棺木登记
- 无遗像
- 祠堂看守人问话

玩家开始改变判断：

```text
问题可能不是谁办葬礼，而是谁被葬礼确认。
```

系统表现：

- 葬礼线可以有效把玩家从传闻线拉回结构层。
- R01 触发自然。
- 玩家开始意识到三场葬礼之间并非独立。

结论：

葬礼线作为结构回拉点有效。

---

### Phase 4：时间轴尝试失败

玩家尝试整理：

```text
第一场葬礼
第二场葬礼
第三场葬礼
十五年前雨夜
近期传闻
纸扎铺夜敲门
```

但发现标准时间顺序无法解释所有碎片。

DM 使用 `anti_timeline_rules.md` 反馈：

```text
你现在排出的可能是某个视角中的顺序，而不一定是底层因果顺序。
```

玩家转而使用：

- `clue_board.md`
- `relationship_map.md`

结论：

防时间轴规则有效。

---

### Phase 5：旧款线进入因果层

玩家调查：

- 半截旧账页
- 修桥碑缺角
- 老路
- 旧麻绳

玩家开始意识到：

```text
旧款线不是背景。
它可能解释为什么旧事必须被压住。
```

系统表现：

- F03、F07、F11、F15 组合有效。
- R03 触发自然。
- 玩家从“故事谜团”进入“因果压力源”。

问题：

旧款线仍需要玩家主动转向地点调查。

---

### Phase 6：最终核验

玩家进入：

- `final_questions.md`
- `final_review_scene.md`
- `case_closure_template.md`

玩家能提交完整结构性结案：

```text
三场葬礼不是三起独立事件。
传闻线是被污染的目击。
纸扎铺线是不完整视角。
旧款线是源头压力。
第三场葬礼是闭环节点。
```

DM 可按 `ending.md` 评分。

结论：

最终核验场景有效。

---

## 4. 测试结果

```text
Run 005：SOFT PASS
```

Case003 可以被真实玩家式路线跑通。

但仍有两个需要优化的问题：

1. 旧款线入口偏晚，可能导致玩家中期在传闻/纸扎铺线停留过久。
2. 前期缺少“地点关系图”，玩家可能不知道哪些地点天然相连。
