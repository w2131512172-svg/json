# Case002 Playtest Plan

案件名：第十九层
状态：DRAFT

---

## 1. Playtest 目标

本文件用于测试 Case002 是否具备可玩性。

测试重点：

1. 玩家是否能从现场异常意识到入室盗窃假象不稳。
2. 玩家是否能通过门锁、监控、证词重建进入路径。
3. 玩家是否能通过法医和电子证据重建死亡时间线。
4. 玩家是否能从现场线转向上游动机线。
5. 玩家是否会过早猜中执行者但忽略主谋。
6. 玩家是否会被误导人物永久卡死。
7. 结案判定是否足够清晰。

---

## 2. 标准测试路线

标准路线不代表唯一解法，只用于验证基础闭环。

```text
开局案件简报
→ 调查玄关与门锁
→ 调查书房与保险柜
→ 调查三名死者位置
→ 申请初步尸检
→ 调查顾念手机与通话记录
→ 调查小区监控和保安登记
→ 询问关键熟人
→ 调查公司线
→ 获取财务/通讯/电子证据
→ 形成结案陈述
```

---

## 3. 测试判定

### PASS

玩家可以在不读取 DM 文件的情况下，通过玩家可见材料形成完整结案链。

### SOFT PASS

玩家能找出主要执行者和案件性质，但主谋或协助者链条不够清楚。

### FAIL

玩家长期卡在错误方向，且现有材料无法自然纠偏。

---

## 4. 已知待测点

- 现场线是否足够吸引玩家继续查门锁。
- 公司线是否过晚出现。
- 误导人物是否太弱或太强。
- 财务证据是否过于直接。
- 法医报告是否足够支撑死亡顺序推理。
