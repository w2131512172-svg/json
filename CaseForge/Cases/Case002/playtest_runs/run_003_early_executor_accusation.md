# Playtest Run 003：过早指控执行者

案件：Case002《第十九层》
状态：DRAFT TEST
路线：玩家较早锁定现场执行者

---

## 1. 测试目的

验证当玩家较早锁定现场执行者时，系统是否能阻止案件过早结束，并推动玩家继续寻找上游动机链。

---

## 2. 模拟玩家行为

玩家通过以下证据较早锁定现场执行者：

- 备用权限。
- 车辆进出记录。
- 不完整不在场证明。
- 体表伤口或微量物证。

玩家提出结案：

```text
我认为本案是熟人入室杀人，现场执行者就是某关键熟人。
```

---

## 3. DM 判定

该结案不应直接 PASS。

原因：

- 可以解释现场执行。
- 不能解释核心失踪物。
- 不能解释案发前诱导。
- 不能解释异常财务来源。
- 不能解释案后误导材料。

---

## 4. 建议反馈

DM 应反馈：

> 你的指控可以解释“谁进入现场”和“谁实施暴力”，但还不能解释“为什么要这么做”。如果只是普通熟人杀人，现场为什么要伪装成盗窃？失踪物品为什么包含特定存储设备？资金和通讯链条又指向哪里？

---

## 5. 推进方向

DM 可以开放或提醒玩家调查：

- 死者工作电脑。
- 失踪外接存储设备。
- 公司近期冲突。
- 案发前通讯记录。
- 异常财务记录。
- 案后主动提供的误导材料。

---

## 6. 测试结论

```text
Run 003：PASS
```

本案允许玩家较早锁定执行者，但结案系统能够阻止“抓到刀就结束”。

这是本案区别于 Case001 的重点：

> 本案不是单点凶手题，而是责任结构题。
